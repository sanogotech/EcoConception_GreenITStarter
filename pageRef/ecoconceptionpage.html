🌿 Image WebP + lazy loading

✔️ Format WebP, dimensions spécifiées, lazy loading, évite CLS.

⚡ JavaScript minimal / defer

Nous utilisons uniquement Bootstrap bundle en defer + quelques lignes légères. Pas de jQuery, pas d'analyseur tiers.

// Aucun script bloquant ; on mesure la performance
document.addEventListener('DOMContentLoaded', () => {
  console.log('Éco page chargée sans surcharge');
});
📖 Explications détaillées des bonnes pratiques (commentaires intégrés)
  • ✅ <link preconnect> et dns-prefetch : anticipation de connexion au CDN Bootstrap, réduit la latence.
  • ✅ Chargement CSS non bloquant : utilisation de media="all" et onload pour ne pas pénaliser le First Paint.
  • ✅ Pas de polices Google Fonts : système par défaut, économise ~300ko de requêtes.
  • ✅ Images optimisées : picture + srcset (WebP prioritaire) loading="lazy" chaque image hors viewport.
  • ✅ DOM réduit : Complexité maîtrisée, pas de milliers de noeuds, évite les réflows.
  • ✅ Pas de carrousel JS ou de librairie inutile → moins d'exécution, moins de mémoire.
  • ✅ `prefers-reduced-motion` : respect des utilisateurs sensibles aux mouvements.
  • ✅ Toutes les images ont width/height explicites pour éviter les changements de layout (CLS).
  • ✅ script defer : exécution après le parsing HTML, non bloquant.
